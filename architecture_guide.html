System Data Flow

An end-to-end overview of the Smart Agriculture 4.0 Ecosystem from physical sensors to AI-driven decisions.

01

IoT Edge Devices

ESP32 microcontrollers deployed in the field collect high-fidelity environmental data including Soil Moisture, NPK levels, Temperature, and Humidity using industrial-grade sensors.

ESP32 Sensors C++
02

MQTT Bridge

Devices transmit JSON payloads wirelessly to the Aedes MQTT Broker. This lightweight protocol ensures reliable communication even in low-bandwidth agricultural environments.

MQTT Aedes Real-time
03

Node.js Central Hub

The central server intercepts sensor messages. It validates data, manages device registration, and prepares the data for the next stage: the AI Brain.

Node.js Express Socket.io
04

Python AI Service

Data is sent to a dedicated Python FastAPI service. Here, Machine Learning models (Scikit-learn, Prophet) analyze stress levels, detect anomalies, and fetch real-world weather data from Open-Meteo.

FastAPI ML Models Open-Meteo
05

Smart Automation

If the AI detects critical stress or predicts severe drying, it issues a Command back through the MQTT bridge to trigger actuators (Pumps, Fertilizer Valves) automatically.

Automation Logic Closed-Loop
06

Next.js Dashboard

The farmer sees everything in real-time. Live charts, the Farm Digital Twin, AI alerts, and PDF reports are hydrated instantly via Socket.io for ultimate situational awareness.

Next.js Tailwind Framer Motion
