# បញ្ហាដែលបានដឹង និងកម្រិតកំណត់នៃប្រព័ន្ធ (Known Issues & Limitations)

ឯកសារនេះរៀបរាប់អំពីកម្រិតកំណត់បច្ចេកទេស និងបញ្ហាសេសសល់មួយចំនួននៅក្នុងប្រព័ន្ធ **Smart Agriculture 4.0** ជំនាន់សាកល្បង (Prototype)។

### ១. ដែនកំណត់នៃ AI Quota (AI Quota & Fallback)
ប្រព័ន្ធប្រើប្រាស់ Gemini API ដើម្បីវិភាគទិន្នន័យ។ ដោយសារការប្រើប្រាស់មានកម្រិត (Rate Limit)៖
- ប្រសិនបើការប្រើប្រាស់ហួសចំណុះ ប្រព័ន្ធនឹងប្តូរទៅប្រើ Logic បែបច្បាប់ទូទៅ (Rule-based) ជាបណ្តោះអាសន្ន។
- ការឆ្លើយតបរបស់ AI អាចមានភាពយឺតយ៉ាវនៅពេលមានសំណើមើលច្រើនក្នុងពេលតែមួយ។

### ២. ការពឹងផ្អែកលើការភ្ជាប់អ៊ីនធឺណិត (Cloud Dependency)
ប្រព័ន្ធនេះជាប្រភេទ Cloud-Native IoT ដូច្នេះវាត្រូវការអ៊ីនធឺណិតជានិច្ច៖
- ការបញ្ជាពីចម្ងាយតាមរយៈ Dashboard មិនអាចធ្វើទៅបានទេប្រសិនបើ ESP32 ឬ Server ដាច់អ៊ីនធឺណិត។
- ទិន្នន័យអាកាសធាតុពឹងផ្អែកទាំងស្រុងលើ OpenWeatherMap API។

### ៣. បរិមាណទិន្នន័យ (Data Volume & Storage)
បច្ចុប្បន្នប្រព័ន្ធបញ្ជូនទិន្នន័យពីសិនស័ររៀងរាល់ **៥ វិនាទី** ម្តង៖
- ក្នុងមួយថ្ងៃមានទិន្នន័យប្រហែល ១៧,២៨០ កំណត់ត្រា (Records) សម្រាប់ឧបករណ៍មួយ។
- ក្នុងរយៈពេលវែង វានឹងត្រូវការការសម្អាតទិន្នន័យចាស់ៗ (Data Pruning) ដើម្បីរក្សាភាពរហ័សនៃ Database។

### ៤. សុវត្ថិភាពនៃពិធីការ MQTT (MQTT Security)
សម្រាប់ជំនាន់សាកល្បងនេះ ការទាក់ទងតាម MQTT មិនទាន់មានការសម្ងាត់កម្រិតខ្ពស់ (TLS/SSL Encryption) ពេញលេញនៅឡើយទេ៖
- ទិន្នន័យបញ្ជា (Command) អាចនឹងត្រូវបានមើលឃើញប្រសិនបើមានការលួចចូលក្នុងបណ្តាញ WiFi តែមួយ។

### ៥. ដែនកំណត់នៃម៉ូឌុលរីឡេ (Actuator Conflict)
ដើម្បីការពារការប្រើប្រាស់ថាមពលលើសកម្រិតរបស់ឧបករណ៍៖
- ប្រព័ន្ធត្រូវបានរចនាឡើងមិនឱ្យបើកម៉ាស៊ីនបូមទឹក និងម៉ាស៊ីនដាក់ជីក្នុងពេលតែមួយឡើយ។
- ការបញ្ជាដោយដៃ (Manual Override) នឹងមានអទិភាពខ្ពស់ជាងការបញ្ជាដោយ AI។
### ៦. ការកែសម្រួលសិនស័រ (Sensor Calibration)
សារធាតុគីមីដែលមាននៅក្នុងសិនស័រប្រភេទ 7-in-1 (ជាពិសេស pH និង EC) អាចនឹងមានលំអៀងតាមពេលវេលា៖
- តម្រូវឱ្យមានការកែសម្រួល (Calibration) ជាមួយសូលុយស្យុងស្តង់ដារ យ៉ាងហោចណាស់ម្តងក្នុងមួយខែ ដើម្បីរក្សាភាពត្រឹមត្រូវខ្ពស់។
- សម្រាប់ Prototype នេះ តម្លៃត្រូវបានកែសម្រួលដំបូងតាមរយៈ Software (Offset mapping) ប៉ុណ្ណោះ។
